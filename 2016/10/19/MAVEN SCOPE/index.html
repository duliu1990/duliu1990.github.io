Maven Scope

Author: Darren Du Date: October 19, 2016  22:10:39 Category: Maven

Maven Scope 目前总共有六种形式：
1.compile
默认范围，在下面所有的classpath都可用。当打war包会包含。

2.provided
只有在当JDK 或者一个容器已提供该依赖之后才使用。例如， 如果你开发了一个web 应用，你可能在编译 classpath 中需要可用的Servlet API 来编译一个servlet，但是你不会想要在打包好的WAR 中包含这个Servlet API；这个Servlet API JAR 由你的应用服务器或者servlet 容器提供。已提供范围的依赖在编译classpath （不是运行时）可用。它们不是传递性的，也不会被打包。

3.runtime
运行和测试系统的时候需要，但在编译的时候不需要。

4.test
在一般的编译和运行时都不需要，它们只有在测试编译和测试运行阶段可用。

5.system
与.provided类似，但是你必须显式的提供一个对于本地系统中JAR 文件的路径。

6.import (2.0.9之后添加)
此范围仅用于在部分类型POM的依赖。这表明指定的POM，应该在POM的部分依赖所代替。由于它们被替换，与import的依赖范围实际上并不参与限制依赖的传递。
